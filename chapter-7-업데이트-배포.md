# 🔄 Chapter 7: "업데이트 배포" (Update Deployment) - 변경사항 세상에 공개하기

> **목표:** 업그레이드한 메모장 앱을 다시 배포해서, 인터넷에 있는 실제 웹사이트도 업데이트합니다.

## Step 7.1: 변경사항을 GitHub에 올리기

지금까지 수정한 코드를 GitHub에 올려야 Vercel이 자동으로 새 버전을 배포해 줍니다.

1. 우측 **채팅창**에:

   > "업데이트한 코드를 GitHub에 올려줘."

2. AI가 다음과 같은 명령어들을 알려줄 겁니다:

   ```bash
   git add .
   git commit -m "Upgrade memo app with title, body, and sidebar"
   git push
   ```

3. 터미널에 명령어들을 **순서대로 복사+붙여넣기** 하세요.

## Step 7.2: Vercel 자동 배포 확인하기

GitHub에 코드를 올리면, Vercel이 자동으로 감지해서 새로운 버전을 배포합니다!

1. [Vercel 대시보드](https://vercel.com/dashboard)로 이동합니다.

2. `my-vibe-canvas` 프로젝트를 클릭합니다.

3. 상단에 **"Building"** 또는 **"Deploying"** 상태가 보일 겁니다. 기다려 주세요!

4. 1~2분 후 **"Ready"** 상태로 바뀌면 배포 완료입니다! 🎉

## Step 7.3: 실제 웹사이트에서 확인하기

1. Vercel 프로젝트 페이지 상단의 **"Visit"** 버튼을 클릭하거나, `your-project.vercel.app` URL로 접속합니다.

2. 업데이트된 기능들이 제대로 작동하나요?
   - ✅ 제목과 본문 입력이 되나요?
   - ✅ 왼쪽 사이드바에 메모 목록이 보이나요?
   - ✅ 메모를 클릭하면 오른쪽에 전체 내용이 나타나나요?
   - ✅ 삭제 버튼이 작동하나요?

3. 친구에게 링크를 다시 보내보세요! 이제 훨씬 더 멋진 메모 앱이 되었습니다.

## 💡 자동 배포의 마법

**중요한 포인트:** 
- 앞으로 코드를 수정할 때마다 `git add . → git commit → git push`만 하면 됩니다.
- Vercel이 자동으로 감지해서 새 버전을 배포해 줍니다!
- 별도로 Vercel 웹사이트에서 뭔가 할 필요가 없습니다.

이게 바로 **"CI/CD (지속적 통합/배포)"**의 시작입니다. 개발자들이 실제로 사용하는 방식이에요!

## 🔧 배포 후 문제가 생겼다면?

만약 배포는 성공했는데, 웹사이트가 이상하게 작동한다면:

1. **브라우저 캐시 삭제**
   - `Ctrl + Shift + R` (Windows) 또는 `Cmd + Shift + R` (Mac)으로 강력 새로고침을 해보세요.

2. **Vercel 로그 확인**
   - Vercel 대시보드에서 최신 배포를 클릭하고 `Runtime Logs`를 확인해 보세요.

3. **AI에게 물어보기**
   - 에러 메시지를 복사해서 Cursor의 우측 채팅창에 붙여넣고 해결 방법을 물어보세요.

## 🤔 하지만 아직 한 가지 문제가...

업그레이드된 메모장 앱, 정말 멋지죠? 하지만 여전히 **localStorage**를 사용하고 있습니다.

이게 뭐가 문제일까요?

- 내 컴퓨터(내 브라우저)에만 메모가 저장됩니다.
- 친구가 내 웹사이트에 접속해도, 내 메모를 볼 수 없습니다.
- 내가 휴대폰으로 접속하면, 내 메모가 보이지 않습니다.
- 브라우저 데이터를 지우면 메모가 모두 사라집니다.

**진짜 서비스처럼 만들려면 어떻게 해야 할까요?** 다음 챕터에서 클라우드 데이터베이스를 사용해 봅시다!

## 🔥 7단계 성과

코드를 수정하고, GitHub에 올리고, 자동으로 전 세계에 배포되는 전체 워크플로우를 경험했습니다! 

이제 여러분은 진짜 개발자처럼 일하고 있습니다! 🚀
