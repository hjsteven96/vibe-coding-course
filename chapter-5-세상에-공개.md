# 🌎 Chapter 5: "세상에 공개" (Deployment) - 전 세계에 자랑하기

> **목표:** 이 웹사이트를 내 친구에게 카톡으로 보낼 수 있는 실제 URL로 만듭니다.

## Step 5.1: GitHub에 코드 올리기

1. 우측 **채팅창**을 엽니다.

   > "깃헙에 올려줘."


2. AI가 `git init`, `git add .`, `git commit...` 등 터미널에 칠 명령어들을 순서대로 알려줄 겁니다. **그대로 복사+붙여넣기** 하세요. (GitHub 로그인 과정이 필요할 수 있습니다.)

## Step 5.2: '사람'이 할 일 (Vercel 배포)

### 💡 배포란?

지금까지 우리는 `localhost:3000`에서만 앱을 볼 수 있었습니다. 이건 '내 컴퓨터 안에서만' 작동하는 주소예요.

**배포(Deployment)**는 이 앱을 인터넷 서버에 올려서, 전 세계 누구나 접속할 수 있는 실제 URL(`xxx.vercel.app` 같은)을 만드는 과정입니다. 

쉽게 말하면: **내 컴퓨터 → 인터넷 서버**로 옮기는 거예요!

---

1. [Vercel 웹사이트](https://vercel.com/)로 가서 GitHub 계정으로 로그인합니다.

2. `Add New Project` 버튼을 누릅니다.

3. 방금 올린 `my-vibe-canvas` 저장소를 찾아 `Import` 버튼을 누릅니다.

4. 설정은 그대로 두고 **`Deploy`** 버튼을 누릅니다.

5. 1~2분 정도 기다리면... 축하 폭죽이 터지며 `my-vibe-canvas.vercel.app` 같은 실제 URL이 생성됩니다!

---

### 배포 오류가 났다면?

배포 중에 빨간색 에러 메시지가 뜬다면 당황하지 마세요! 다음 순서대로 해결해 봅시다:

1. **에러 로그 확인하기**
   - Vercel 화면에서 `View Build Logs` 또는 `View Deployment`를 클릭합니다.
   - 빨간색으로 표시된 에러 메시지를 찾아 **전체 복사**합니다.

2. **AI에게 물어보기**
   - Cursor로 돌아와 우측 **채팅창**에 이렇게 물어보세요:
   
   > "Vercel 배포하는데 이런 에러가 났어:
   > 
   > (여기에 에러 메시지 붙여넣기)
   > 
   > 해결해줘"

3. **AI가 해결책을 알려줄 겁니다.** 주로 다음과 같은 문제들입니다:
   - 패키지 설치 오류 → `package.json` 수정 필요
   - 빌드 에러 → 코드에서 오타나 문법 오류 수정
   - 환경 설정 문제 → Next.js 설정 파일 조정

4. **코드를 수정한 후 다시 GitHub에 올리기**
   - 터미널에서:
   ```bash
   git add .
   git commit -m "Fix deployment error"
   git push
   ```
   - Vercel이 자동으로 다시 배포를 시도합니다!

5. **그래도 안 된다면?**
   - 에러 메시지를 구글에 검색해 보세요. (영어로 검색하면 더 많은 정보가 나옵니다)
   - Vercel 대시보드에서 `Redeploy` 버튼을 눌러 재배포를 시도해 보세요.

---

## Step 5.3: 테스트해보기

1. Vercel이 준 URL을 클릭해서 열어보세요.

2. 메모를 몇 개 추가하고, 새로고침도 해보세요. 잘 작동하나요?

3. 이 링크를 친구에게 공유해 보세요! 이제 여러분은 실제 웹 서비스를 런칭한 겁니다! 🎉

## 💡 알아두면 좋은 것

- **무료입니다:** Vercel은 개인 프로젝트에 대해서는 완전 무료입니다.
- **자동 배포:** 앞으로 GitHub에 코드를 수정해서 올리면, Vercel이 자동으로 새 버전을 배포해 줍니다.
- **HTTPS 기본:** 보안 인증서(HTTPS)가 자동으로 적용됩니다.

## 🤔 개선할 점이 보이나요?

지금 만든 메모장 앱도 훌륭하지만, 더 개선할 수 있는 부분들이 있습니다:

- 메모가 한 줄만 보여서, 어떤 내용인지 미리 보기가 어렵습니다.
- 제목과 본문을 구분하면 더 정리가 잘 될 것 같습니다.
- 메모 목록과 상세 내용을 나눠서 보여주면 더 편할 것 같습니다.
